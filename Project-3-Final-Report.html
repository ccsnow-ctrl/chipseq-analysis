INTRODUCTION

The RUNX family of transcription factors includes three members: RUNX1, RUNX2, and RUNX3. The RUNX family are regulators of multiple phenotypes as well as many signalling pathways which allow them to play a significant role in the development of many cancers. Since RUNX1 is involved in breast cancer and genome organization is crucial for proper cell function, it's important to understand how RUNX1 influences chromatin structure and what role this plays in breast cancer development.Researchers in the original study wanted to gain new understanding of how RUNX1 causes changes to the way genomic DNA is organized and how these structural changes relate to the altered gene expression patterns seen in breast cancer cells. To investigate this, they combined three different bioinformatics techniques to get a complete picture of RUNX1's role. Hi-C was used to map out how different parts of chromosomes interact with each other and to detect changes in chromatin structure. RNA-seq helped measure which genes were turned on or off in response to RUNX1 suppression. Finally, ChIP-seq identified exactly where RUNX1 binds to the genome, providing insight into how it regulates chromatin organization. Together, these techniques allowed them to connect RUNX1's binding locations with the changes they observed in both genome structure and gene activity.

METHODS

This ChIP-seq data analysis was performed using a Nextflow pipeline which began with several input channels including raw sequencing reads, adapter sequences, a BED file for genomic regions, reference genome files, a blacklist file for filtering artifact regions and a GTF annotation file. The first thing we wanted to do was an initial quality control assessment, for which we used FastQC v.0.12.1, which generated reports on quality metrics, GC content, and potential contamination. At the same time, reads were processed through Trimmomatic v.0.39 for adapter removal using the provided adapter sequences and quality thresholds. The quality control metrics from FastQC, Trimmomatic, and SAMtools Flagstat were combined using MultiQC v. 1.25 to generate a comprehensive quality report.

The next thing we focused on was aligning our results to the reference genome. We built the reference genome index using Bowtie2 v.2.5.4. Our trimmed reads were then aligned to the reference genome using Bowtie2 v.2.5.4, which mapped the sequences while accounting for mismatches and gaps. The resulting BAM files were sorted using SAMtools v.1.21, followed by indexing with SAMtools.

From the sorted and indexed BAM files, normalized coverage tracks were generated using bamCoverage from deepTools v.3.5.5, which converted the alignments into bigWig files. With these BigWig files we used computeMatrix from deepTools to calculate enrichment scores across specified genomic regions and plotProfile from deepTools to visualize the average signal distribution as line plots. We also took the BigWig files and ran them through multiBigwigSummary from deepTools to compute pairwise scores across all samples, which were then used by plotCorrelation from deepTools to generate correlation heatmaps and scatter plots assessing sample similarity and replicate reproducibility.

A separate analysis branch took our aligned reads and ran them through HOMER tools for peak identification and characterization. The sorted BAM files and genome information were processed by makeTagDirectory to create tag directories that organize reads for efficient peak calling. findPeaks then identified regions of significant ChIP-seq enrichment using local Poisson background models. The resulting peak files were converted to BED format using pos2bed for compatibility with downstream tools. Intersect from bedtools v2.31.1 identified overlapping peaks between samples or with the input BED file to find consensus binding sites. Remove filtered out peaks by excluding regions listed in the blacklist file. The filtered peaks were then processed by annotatePeaks.pl from HOMER, which assigned peaks to their nearest genes and genomic features using the GTF annotation file. Finally, findMotifsGenome.pl from HOMER performed de novo motif discovery and known motif enrichment analysis within the peak sequences to identify transcription factor binding motifs, using the reference genome as background.

The pipeline used in this study generated multiple output files including quality control reports, alignment statistics, normalized coverage tracks, peak lists, annotated peak-gene associations, and motif enrichment results, providing a comprehensive analysis of the ChIP-seq experiment.

INPUT_rep2 had the lowest read count at around 10.1 million, while INPUT_rep1 had 28.6M, IP_rep1 had 27.8M, and IP_rep2 had 28.2M. The only notable finding was the presence of overrepresented adapter sequences, specifically GATCGGAAGAGCACACGTCTGAACTCCAGTCACTCCCGAATCTCGTATGC and GATCGGAAGAGCACACGTCTGAACTCCAGTCACTCGAAGATCTCGTATGC, which is typical and expected for Illumina libraries prior to trimming. Alignment metrics were strong across all samples, with each showing greater than 80–90% alignment to the reference genome, indicating that the data mapped well and the sequencing libraries were generally high quality.

Signal Coverage

The signal coverage plots for IP_rep1 and IP_rep2 show the average ChIP-seq read density across all annotated genes, aligned by their transcription start sites (TSS) and transcription end sites (TES). In both replicates, there is a very strong and sharp enrichment peak exactly at the TSS, followed by a rapid drop in signal and a relatively flat profile across the rest of the gene body. This pattern indicates that the immunoprecipitated factor is strongly localized to promoter regions rather than gene bodies, which is typical for many transcription-related proteins. Importantly, the two replicates display nearly identical shapes and signal levels, demonstrating strong reproducibility of the experiment and confirming that the ChIP enrichment successfully captured real biological binding patterns.

def load_bed_intervals(path):
    """Load BED file as dictionary of chromosome -> list of (start, end) tuples"""
    intervals = {}
    with open(path, "r") as f:
        for line in f:
            if line.startswith("#") or line.strip() == "":
                continue
            parts = line.strip().split()
            if len(parts) < 3:
                continue
            chrom = parts[0]
            start = int(parts[1])
            end = int(parts[2])
            if chrom not in intervals:
                intervals[chrom] = []
            intervals[chrom].append((start, end))
    return intervals

def intervals_overlap(a_start, a_end, b_start, b_end):
    """Check if two intervals overlap"""
    return not (a_end <= b_start or b_end <= a_start)

def count_peak_overlaps(rep1_intervals, rep2_intervals):
    """Count overlapping peaks between two replicates"""
    rep1_only = 0
    rep2_only = 0
    shared = 0
    
    # Get all chromosomes
    all_chroms = set(rep1_intervals.keys()) | set(rep2_intervals.keys())
    
    for chrom in all_chroms:
        rep1_peaks = rep1_intervals.get(chrom, [])
        rep2_peaks = rep2_intervals.get(chrom, [])
        
        # Track which rep2 peaks have been matched
        rep2_matched = [False] * len(rep2_peaks)
        
        # For each rep1 peak, check if it overlaps any rep2 peak
        for r1_start, r1_end in rep1_peaks:
            found_overlap = False
            for i, (r2_start, r2_end) in enumerate(rep2_peaks):
                if intervals_overlap(r1_start, r1_end, r2_start, r2_end):
                    shared += 1
                    rep2_matched[i] = True
                    found_overlap = True
                    break  # Count each rep1 peak only once
            
            if not found_overlap:
                rep1_only += 1
        
        # Count rep2 peaks that didn't match any rep1 peak
        rep2_only += sum(1 for matched in rep2_matched if not matched)
    
    return rep1_only, rep2_only, shared
